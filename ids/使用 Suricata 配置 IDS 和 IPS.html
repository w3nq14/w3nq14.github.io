使用 Suricata 配置 IDS 和 IPS

本文档详细介绍了如何在基于 Linux 的环境下，将 Suricata 配置为入侵检测系统 (IDS) 和入侵防御系统 (IPS) 。配置步骤包括规则配置、测试、攻击模拟和流量分析。

攻击机使用：Kali Linux

目标机器：Ubuntu

第一部分：将 Suricata 配置为入侵检测系统

  1. 安装 Meerkat

使用相应的软件包管理器在目标 Ubuntu 系统上安装 Suricata。命令：sudo apt install suricata -y

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/11ywXqIsC9nmcuWrMtM5APA.png)

2. 配置检测规则

打开位于 /etc/suricata/suricata.yaml 的 Suricata 配置文件。命令：sudo nano /etc/suricata/suricata.yaml

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1ucWZ9xYKx7bZ7OEzoXnm8w.png)

3. 确定系统 IP 地址

打开一个新的终端窗口，执行以下命令来获取系统的 IP 地址：ip a

使用已识别的 IP 地址更新 suricata.yaml 文件中的 HOME_NET 字段。

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1lyTpc6DCa4hPx7yEwy7bJw.png)

4. 配置网络接口监控

在 suricata.yaml 文件中：

  1. 搜索 af-packet 配置（Ctrl + W）。
  2. 设置要监控的适当网络接口（例如，eth0）。

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1-AGJQHsORJoWrSL6fnMsgw.png)

5. 配置规则文件

在配置文件中查找规则文件，并将默认值替换为：local.rules

如果文件不存在，则会在下一步创建该文件。

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1xpXwy3OV8fLSoFmVtSMU5g.png)

6. 创建和添加自定义规则

使用以下命令创建本地规则文件：sudo nano /var/lib/suricata/rules/local.rules

添加以下 ICMP 洪水攻击检测规则：

规则 1：中等程度的 ICMP 洪水检测

alert icmp any any -> $HOME_NET any \

（msg:”警报：检测到 ICMP 洪水攻击。5 秒内发送了超过 5 个数据包”；

阈值：类型限制，按源跟踪，计数 5，秒 5；sid：1000001；）

规则 2：积极主动的 ICMP 洪水攻击检测

alert icmp any any -> $HOME_NET any \

（消息：“严重：检测到恶意 ICMP 洪水攻击。3 秒内发送了超过 20 个数据包”；

阈值：类型限制，按源跟踪，计数 20，秒 3；sid：1000003；）

使用以下命令验证规则配置：

sudo suricata -T -c /etc/suricata/suricata.yaml -v

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1zdH3O-pB9aYe6idDYEEUQA.png)

7. 重启并验证 Suricata 服务

重启 Suricata 并检查其状态：

sudo systemctl restart suricata

sudo systemctl status suricata

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1BuuTnmfty69p0KGd__FULQ.png)

8. 监控 Suricata 日志

要分析 Suricata 生成的警报，请监控快速日志文件：

sudo tail -f /var/log/suricata/fast.log

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1eFWQsja7jXhMpabSoCd3GA.png)

9. 模拟 ICMP 拒绝服务攻击

在攻击者机器（Kali Linux）上，执行以下命令：

sudo hping3 -1 -i u2500 -c 100

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1_PzKh4gXciTK6K4kXQ90dg.png)

现在应该可以在目标机器日志中看到生成的警报，这证实了入侵检测系统 (IDS) 检测成功。

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1NS5U0WTOIs5Se6FAi1lYfg.png)

第二部分：将 Suricata 配置为 IPS

1. 修改 Suricata 配置

打开 suricata.yaml 文件，并使用 # 注释掉 af-packet 配置，禁用它。

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1_4GfbervSTJciRzrcYPirw.png)

2. 启用 NFQUEUE 模式

添加以下 NFQUEUE 配置：

nfqueue：

- id：0

接受标记：1

故障开启：是

批次数量：20

模式：接受

线程：自动

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1ci6_kqik2K_z5JzL-dWgGw.png)

3. 更新 Suricata 服务模式

编辑 Suricata 服务文件：

sudo systemctl edit –full suricata.service

将执行模式更新为 NFQUEUE，方法是添加：

  • q 0

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1oKUlBJU614etWVMzKuFj4Q.png)

4. 配置 IPS 丢弃规则

将以下 IPS 规则添加到 local.rules 文件中：

drop icmp any any -> $HOME_NET any \

（消息：“ICMP 丢弃：速率超出限制（3 秒内发送了 3 个数据包）”；

itype:8; detection_filter:track by_dst, count 3, seconds 3; \

sid:1000004; rev:1;)

在继续操作之前，请使用 Suricata 测试命令测试规则配置。

5. 为 NFQUEUE 配置 IPTables

清除现有的 IPTables 规则：

sudo iptables -F

将流量重定向到 NFQUEUE：

sudo iptables -I INPUT -j NFQUEUE - queue-num 0

sudo iptables -I OUTPUT -j NFQUEUE –queue-num 0

sudo iptables -I FORWARD -j NFQUEUE — queue-num 0

检查 IPTables 配置：

sudo iptables -L -v

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1LzIjh57g7B8uGFFPfDiM2w.png)

6. 重新加载并重启服务

重新加载系统配置并重启 Suricata：

sudo systemctl daemon-reload

sudo systemctl restart suricata

7. 验证 IPS 功能

监控 Suricata 日志文件：

sudo tail -f /var/log/suricata/fast.log

从攻击者机器再次发起 ICMP 攻击：

sudo hping3 -1 -i u2500 -c 100

Suricata 现在应该主动丢弃 ICMP 数据包，一旦超过阈值，即可阻止攻击。

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1y2m-9B6nnA8wKMsl4TLXkw.png)

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1MTjJVdqOddCCpszA7f3wNw.png)

交通分析

Wireshark 可以与 Suricata 日志配合使用，实时捕获和分析网络流量。通过丢包统计和捕获的帧，可以清晰地观察到丢包和 ICMP 洪水攻击缓解情况。

Press enter or click to view image in full size

![img](./使用 Suricata 配置 IDS 和 IPS.assets/1A9jJF4ye6KuPsUmLuqvIXA.png)

结论

此配置成功演示了如何将 Suricata 同时部署为入侵检测系统 (IDS) 和入侵防御系统 (IPS)。IDS 可检测可疑的 ICMP 洪水攻击活动，而 IPS 则利用 NFQUEUE 和 IPTables 集成主动拦截恶意流量，从而增强网络安全。
